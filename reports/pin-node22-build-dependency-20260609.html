Pin Node LTS for LingTai Homebrew/Linuxbrew Builds

This tap fix changes the build-time Node dependency from moving latest node to current LTS node@22.

Why

Jason's Linuxbrew upgrade from lingtai-tui 0.8.14 to 0.8.15 failed while the formula ran portal/web/npm ci. The failing binary was Linuxbrew's latest node 26.3.0, which required newer host ABI symbols than the target machine provided:

GLIBC_2.38 GLIBCXX_3.4.31 / GLIBCXX_3.4.32 GCC_13.0.0

The formula's source-build flow is fine; the problem is depending on Homebrew's moving latest Node line for that source build.

What changed

  • depends_on "node" => :build becomes depends_on "node@22" => :build.
  • The formula prepends Formula["node@22"].opt_bin to PATH during install so npm/node come from the pinned LTS keg even if latest Node is installed.
  • The portal frontend still builds locally with npm ci && npm run build.

Evidence

  • node@22 is not deprecated and is the current LTS formula; Homebrew metadata shows deprecation date 2026-10-28.
  • node@20 is already deprecated, so it is less suitable as the default.
  • Inspection of the x86_64 Linux node@22 bottle found required symbols up to GLIBC_2.34 / GLIBCXX_3.4.30, lower than the failing node 26 requirement reported by Linuxbrew.

Validation

  • Ruby syntax check for lingtai-tui.rb.
  • Formula diff reviewed manually.
  • Full Linuxbrew validation still requires retrying on the affected Linux host after the tap PR/commit is available.
